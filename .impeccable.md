# Sketch Forge — Design Context

## Users
Broad: students, thinkers, designers, indie makers — anyone who reaches for paper when they're trying to figure something out. Used at a desk, on a tablet during a meeting, on a phone in transit. The job: turn messy thinking into something you can come back to.

## Brand Personality
Honest · hand-made · considered. Quietly confident. The product is a tool, not a platform. It sounds like a designer who reads, not a startup that ships.

Emotional goals: calm, crafted, honest. The interface should reward attention without demanding it.

## Aesthetic Direction
**Editorial / paper-craft.** Layout reads like a small-format design zine or a thoughtful magazine spread — strong typographic hierarchy, asymmetric grids, marginalia, drop caps, rule lines, numbered sections. The handwritten accent (Kalam) is reserved for captions, marginalia, and section numerals — used like a designer would annotate a printed proof.

- **Theme:** light cream default (reads as paper). Dark warm theme remains toggleable.
- **Typography:** Spectral for editorial display + body serif passages. Geist Sans for UI/labels. Kalam for handwritten marginalia only — never for body.
- **Anti-references:** generic SaaS hero with gradient text, three-up icon-card features, glassmorphism, cyan-on-dark.

## Design Principles
1. **Paper, not glass.** Surfaces feel like stock, not screens. No blur, no glow as decoration.
2. **Type does the work.** Hierarchy comes from size and weight contrast, not boxes.
3. **Marginalia, not decoration.** Hand-drawn elements appear where they'd appear in a real editor's proof — captions, asides, arrows that point at things.
4. **Asymmetry over symmetry.** Left-aligned body, generous gutters, unequal columns. Center-aligned only for short editorial moments.
5. **Restraint with the accent.** Gold/amber accent is rare — used like a highlighter pass on a printed page.
